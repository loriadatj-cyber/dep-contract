# Security policy

## Supported versions

Security fixes are provided for the latest released major version.

## Reporting

Do not open a public issue for a vulnerability that could put users at risk.
Use GitHub private vulnerability reporting on this repository instead.

Include affected versions, impact, reproduction steps, and any suggested
mitigation. Please remove registry credentials and private package names from
reports and fixtures.

The maintainers aim to acknowledge a report within 7 days and provide an
initial assessment within 14 days.
